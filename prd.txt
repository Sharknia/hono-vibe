# 프로젝트 요구사항 정의서 (PRD): Hono Vibe 크롤링 및 알림 기능

## 1. 프로젝트 개요

사용자가 지정한 키워드에 대한 '핫딜' 정보를 여러 웹사이트에서 크롤링하여, 해당 정보를 이메일로 알림을 보내주는 시스템을 구축한다. 이 시스템은 확장성을 고려하여 새로운 크롤링 대상 사이트를 쉽게 추가할 수 있는 구조를 가지며, 관리자가 사용자 및 키워드를 효율적으로 관리할 수 있는 기능을 제공한다.

## 2. 사용자 페르소나

-   **일반 사용자:** 특정 상품의 '핫딜' 정보에 관심이 많으며, 원하는 키워드를 등록하고 관련 정보를 이메일로 받아보기를 원한다.
-   **관리자:** 시스템의 전반적인 운영을 담당하며, 사용자 계정 및 전체 크롤링 키워드를 관리할 책임이 있다.

## 3. 주요 기능 요구사항

### 3.1. 크롤링 시스템
-   지정된 여러 웹사이트의 게시판 또는 상품 목록을 주기적으로 크롤링한다.
-   **(확장성)** 전략 패턴(Strategy Pattern)을 적용하여 새로운 크롤링 대상 사이트를 최소한의 코드 수정으로 쉽게 추가/수정/삭제할 수 있도록 설계한다.
-   **(성능)** 여러 사이트를 동시에 효율적으로 크롤링할 수 있도록 병렬 처리 기능을 구현한다.

### 3.2. 이메일 알림 시스템
-   크롤링 결과, 사용자가 구독한 키워드와 일치하는 새로운 '핫딜' 정보가 감지되면 해당 사용자에게 이메일을 발송한다.
-   **(성능)** 다수의 사용자에게 이메일을 보낼 때 병렬 처리를 통해 신속하게 발송한다.

### 3.3. 사용자 기능 API
-   **키워드 관리:**
    -   전체 키워드 목록 조회
    -   사용자가 원하는 키워드를 구독(등록)
    -   사용자가 구독한 키워드 목록 조회
    -   구독한 키워드 삭제
-   **조회 기능:**
    -   특정 키워드에 대한 최신 크롤링 결과(핫딜 정보) 조회

### 3.4. 관리자 기능 API
-   **사용자 관리:**
    -   전체 사용자 목록 조회
    -   사용자 정보 수정 및 삭제
-   **키워드 관리:**
    -   시스템에 등록된 전체 키워드 목록 조회, 추가, 삭제
    -   특정 사용자가 구독 중인 키워드 목록 조회

## 4. 데이터베이스 스키마 설계 (제안)

-   **Users:** 사용자 정보
    -   `id` (PK)
    -   `email` (Unique)
    -   `password`
    -   `nickname`
    -   `role` ('user', 'admin')
    -   `createdAt`
    -   `updatedAt`

-   **Keywords:** 전체 키워드 목록
    -   `id` (PK)
    -   `name` (Unique): 키워드명 (e.g., "RTX 4090")
    -   `createdAt`

-   **UserKeywords:** 사용자와 키워드의 구독 관계 (Many-to-Many)
    -   `userId` (FK, PK)
    -   `keywordId` (FK, PK)

-   **CrawlingSites:** 크롤링 대상 사이트 정보
    -   `id` (PK)
    -   `name`: 사이트 이름 (e.g., "퀘이사존")
    -   `baseUrl`: 사이트 기본 URL
    -   `strategy`: 해당 사이트 크롤링에 사용할 전략 식별자

-   **CrawledResults:** 크롤링 결과 데이터
    -   `id` (PK)
    -   `keywordId` (FK): 어떤 키워드로 검색된 결과인지
    -   `siteId` (FK): 어느 사이트에서 크롤링되었는지
    -   `title`: 게시글 또는 상품명
    -   `url`: 상세 페이지 URL
    -   `price`: 가격 (선택적)
    -   `crawledAt`: 크롤링된 시간

## 5. API 엔드포인트 명세 (제안)

### 5.1. 사용자 API (`/api`)
-   `GET /keywords`: 구독 가능한 전체 키워드 목록 조회
-   `GET /me/keywords`: 내가 구독한 키워드 목록 조회
-   `POST /me/keywords`: 키워드 구독 (Request Body: `{ "keywordId": 1 }`)
-   `DELETE /me/keywords/:keywordId`: 구독한 키워드 삭제
-   `GET /keywords/:keywordId/results`: 특정 키워드의 최신 핫딜 결과 조회

### 5.2. 관리자 API (`/api/admin`)
-   `GET /users`: 전체 유저 목록 조회
-   `GET /users/:userId`: 특정 유저 정보 조회
-   `GET /keywords`: 전체 키워드 리스트 조회
-   `POST /keywords`: 새 키워드 추가 (Request Body: `{ "name": "새키워드" }`)
-   `PUT /keywords/:keywordId`: 키워드 수정
-   `DELETE /keywords/:keywordId`: 키워드 삭제
